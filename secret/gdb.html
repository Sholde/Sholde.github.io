1 Introduction

gdb is the GNU debugger. It is often used to debug ELF programs written in C.

2 Installation

gdb is a popular tool and it is probably in your pakage manager list.

2.1 Exemple on Arch Linux

$ sudo pacman -S --needed gdb

3 Usage

Compile your program normally. Then run this command to use gdb: 

$ gdb ./program

4 Interface

gdb is a mini-shell, that is it will use your terminal but you will be able to type only its comand.

5 List of command

5.1 Execute and quit

  • r or run: run your program in gdb
  • q or quit: quit gdb

5.2 Breakpoint

  • b or breakpoint: set a breakpoint
  • c or continue: continue to the next breakpoint

5.3 Print

  • p or print: print the value of the variable
  • w or watch: print the value of the variable each time where it is set

5.4 Callstak

  • up: go back to call
  • bt: print the callstack

5.5 Disassemble

  • disas or disassemble: disassemble the given ot the latest function where breakpoint is set

5.6 Move

  • stepi or nexti: they will go at the next assembler instruction but nexti will just go on your program instuction normally.
  • s or step / n or next: same as above but here they will go to the next code instruciton

6 Tips

Use -g3 flag at compile line to enable all gdb fonctionality.

7 Exemple

#include <stdio.h>

int main(int argc, char **argv)
{
  char *s = "Hello World";
  printf("%s\n", s);
  return 0;
}

$ gcc -g3 hello.c -o hello

$ gdb ./hello
